# Agent Audit: Scope

Date: YYYY-MM-DD
Session: Link or identifier

## Expected Behavior

State the objective, constraints, and expected evidence.

## Observed Behavior

Describe actions and outcomes using transcript, command, diff, and test evidence.

## Effective Patterns

Identify behavior worth preserving.

## Failures and Deviations

Identify incorrect assumptions, scope drift, missing verification, repeated
work, excessive context, or unsafe actions.

## Root Causes

Separate instruction gaps, tool limitations, environment failures, model
variation, and user ambiguity.

## Improvements

Recommend the smallest change to prompts, documentation, skills, tooling, or
workflow. Capability changes require a RED–GREEN evaluation scenario.

## Token Efficiency

Assess duplicated reads, unnecessary searches, excessive subagents, avoidable
compaction, and whether a cheaper model or deterministic script was suitable.
